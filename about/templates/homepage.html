WELCOME

I'm a recent graduate who has experience working with a range of Open Source technologies to build responsive and highly scalable web applications.

For the last two years I've been working for CGI, where I have contributed to their application management platform by writing idiomatic Python and presenting to prospective clients. Outside of my project work, I've helped to organise both Python and Red Hat training sessions for other CGI members, and represented CGI at PyConUK.

In my own time I've built a number of websites using the Python web framework django, including a website for the textile designer Emma Golding and my own personal blog. This has given me the freedom to experiment with new technologies and improve my front end skillset.

My family home is in Bristol, I spent three years at the University of Leeds, and I currently live Reading. In my spare time I enjoy walking my family dog, playing sports and pretending to be a DJ.

PORTFOLIO

Server Side

For the last two years I've been writing code for an application using a LAPP stack. This has given me exposure to Red Hat Enterprise Linux, Apache HTTP Server, PostgreSQL and Python. I'm particularly familiar with the later, and have enjoyed sharing this knowledge with others by delivering Python training sessions across the UK. In the near future I hope to start working towards RHEL System Administration accreditation, to build upon the existing skills I have aquired when working on Ubuntu and RHEL systems. I'm also interested in NoSQL databases, and have had some exposure to Cassandra through my project work.

Client Side

I'm no stranger to server side technologies either, and have a good working knowledge of HTML, CSS and JavaScript. I've used these skills to build responsive, mobile friendly and intuitive user iterfaces - and take great pride in my work. In addition to these core scripting languages, I often use jQuery to speed up the development workflow, AJAX to improve the user experience, and have recently started to utilise LESS to make my style sheets more maintainable.

Frameworks

The application management and collaboration tool I have worked on for the last two years is a fork of the Trac framework. Consequently I'm very familiar with the backend design and implementation of Trac, and have written a number of plugins to extend its functionality. This includes burn down chart, notification system and project template plugins, as well as a number of wiki macros to improve the user experience. I'm also familiar with the Django web framework, which I've used to build a number of websites, and the front end Twitter Bootstrap framework, which I've also used in both a professional capacity, and for my own side projects.

Tools

I use both Subversion and Git on a daily basis, and believe VCS are best practice often undervalued. When working with Subversion repositories, I usually utilise Git-SVN to bridge the gap between the two technologies and increase the functionality available. Sublime-Text is also my editor of choice, and I use a range of packages to maximise my productivity as a developer.
